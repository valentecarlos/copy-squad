### Workflow file canônico

> **Source:** Architecture §5.2

```yaml
# .github/workflows/validate.yml
name: validate

on:
  pull_request:
    branches: [main]
  push:
    branches: ['**']

permissions:
  contents: read

jobs:
  validate-structure:
    name: Validate YAML structure
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: node scripts/validate-structure.js

  validate-density:
    name: Validate density (NFR1 + NFR2)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: node scripts/validate-density.js

  encoding-check:
    name: UTF-8 + LF + no BOM
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Detect BOM
        run: |
          if grep -rlI $'\xEF\xBB\xBF' . 2>/dev/null; then
            echo "BOM detected — failing CI"
            exit 1
          fi
      - name: Detect CRLF
        run: |
          if grep -rlI $'\r$' .claude/ research/ docs/ scripts/ tests/ 2>/dev/null; then
            echo "CRLF line endings detected — failing CI"
            exit 1
          fi
```

### Encoding edge case — gitattributes vs. arquivos existentes

Após adicionar `.gitattributes`, git pode tentar normalizar arquivos existentes em commit subsequente. Mitigação:
- Validar com `git status` após adicionar `.gitattributes` que nenhum arquivo foi modificado pelo git auto-normalization
- Se houver normalization, fazer commit separado documentando: `chore: normalize line endings via .gitattributes`
