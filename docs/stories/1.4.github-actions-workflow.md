# Story 1.4: GitHub Actions workflow

## Status

**Draft**

## Story

**As a** maintainer,
**I want** CI workflow `.github/workflows/validate.yml` rodando os scripts de validação em todo PR contra `main` e push em qualquer branch,
**so that** nenhum PR mergea com agente ou dossier abaixo dos critérios — gates automáticos substituem manual review.

## Acceptance Criteria

1. Arquivo `.github/workflows/validate.yml` criado
2. Trigger: `pull_request` contra `main` + `push` em qualquer branch
3. Jobs executam em `ubuntu-latest` (paralelos)
4. Setup Node.js 20+, `npm ci` para instalar deps (`js-yaml`)
5. Job `validate-density` roda `node scripts/validate-density.js` (sem `--report-only`)
6. Job `validate-structure` roda `node scripts/validate-structure.js`
7. Job `encoding-check` valida UTF-8 + LF + sem BOM em todos os arquivos texto
8. Workflow do próprio PR que cria o arquivo passa em verde (rodando contra repo atual: 0 agents, 0 dossiers — 0 violations, exit 0)
9. `.gitattributes` criado força LF cross-platform (Architecture §3.8 defesa em camadas)
10. `.editorconfig` criado força UTF-8 + LF + 2-space indent (Architecture §3.8)

### Workflow file canônico

> **Source:** Architecture §5.2

```yaml
# .github/workflows/validate.yml
name: validate

on:
  pull_request:
    branches: [main]
  push:
    branches: ['**']

permissions:
  contents: read

jobs:
  validate-structure:
    name: Validate YAML structure
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: node scripts/validate-structure.js

  validate-density:
    name: Validate density (NFR1 + NFR2)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: node scripts/validate-density.js

  encoding-check:
    name: UTF-8 + LF + no BOM
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Detect BOM
        run: |
          if grep -rlI $'\xEF\xBB\xBF' . 2>/dev/null; then
            echo "BOM detected — failing CI"
            exit 1
          fi
      - name: Detect CRLF
        run: |
          if grep -rlI $'\r$' .claude/ research/ docs/ scripts/ tests/ 2>/dev/null; then
            echo "CRLF line endings detected — failing CI"
            exit 1
          fi
```

### Encoding edge case — gitattributes vs. arquivos existentes

Após adicionar `.gitattributes`, git pode tentar normalizar arquivos existentes em commit subsequente. Mitigação:
- Validar com `git status` após adicionar `.gitattributes` que nenhum arquivo foi modificado pelo git auto-normalization
- Se houver normalization, fazer commit separado documentando: `chore: normalize line endings via .gitattributes`

### Performance considerations

- npm cache via `cache: 'npm'` em setup-node@v4 — após primeira run, instalações <10s

### Technical Constraints

- **Encoding** UTF-8 + LF em todos os arquivos novos (validation manual antes do commit)
- **GitHub Actions versions** pinadas: `actions/checkout@v4`, `actions/setup-node@v4`, `actions/upload-artifact@v4`
