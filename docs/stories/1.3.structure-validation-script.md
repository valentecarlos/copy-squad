# Story 1.3: Structure validation script

## Status

**Ready**

> **Validated by @po (Pax) on 2026-04-26** — 10/10 score, 0 should-fix. Verdict: **GO**.

## Story

**As a** maintainer,
**I want** script `scripts/validate-structure.js` que valida YAML frontmatter dos agentes contra schema canônico,
**so that** erros estruturais (campos faltando, names duplicados, tools inválidas, type mismatch, FR5 violation) sejam pegos antes do merge — sem deixar passar para review humano.

## Acceptance Criteria

1. Script executável via `node scripts/validate-structure.js`
2. Parsea YAML frontmatter de cada `.claude/agents/*.md` (incluindo `_shared/`)
3. Valida campos obrigatórios presentes: `name`, `description`, `tools`, `type`, `specialty`, `research_path`, `language`
4. Valida `name` único entre todos os agentes (sem colisões)
5. Valida `tools` contém apenas tools da matriz canônica por `type` (Architecture §4.2): orchestrator/copywriter/researcher/reviewer
6. Valida que cada agente referencia pelo menos 1 vez `research/{name}/` no body (FR5)
7. Falha com exit 1 em qualquer violação, com mensagem clara apontando arquivo + violação específica
8. Roda contra fixtures de teste em `tests/fixtures/structure/` cobrindo: agente válido, agente sem `name`, agente com `tools` inválida, agente sem ref a research/, 2 agentes com mesmo name (collision), agente non-orchestrator com `Agent` tool

### Schema canônico de validation

> **Source:** `docs/architecture.md#4.2-matriz-canônica-dec-003` + §3.2 Agent File Format

```javascript
const REQUIRED_FIELDS = [
  'name',          // kebab-case unique
  'description',   // multiline
  'tools',         // array
  'type',          // enum
  'specialty',     // enum
  'research_path', // path string
  'language',      // BCP-47 code (pt-BR)
];

const VALID_TYPES = ['orchestrator', 'copywriter', 'researcher', 'reviewer'];

const VALID_SPECIALTIES = ['direct-response', 'brand', 'orchestrator', 'support'];

const ALLOWED_TOOLS_BY_TYPE = {
  orchestrator: ['Read', 'Grep', 'Glob', 'Agent', 'TodoWrite'],
  copywriter:   ['Read', 'Grep'],
  researcher:   ['Read', 'Grep', 'Write', 'WebSearch', 'WebFetch'],
  reviewer:     ['Read', 'Grep'],
};
```
