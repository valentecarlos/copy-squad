# Story 1.5: Shared frameworks knowledge base

## Status

**Draft**

## Story

**As a** copywriter agent,
**I want** documento `.claude/agents/_shared/frameworks.md` com 8 frameworks consolidados de copywriting (AIDA, PAS, 4 U's, awareness levels, sophistication, slippery slide, inherent drama, reason-why),
**so that** eu (e os outros 14 copywriters) posso referenciá-lo sem duplicar conteúdo em cada agent file — knowledge base compartilhada DRY.

## Acceptance Criteria

1. Arquivo `.claude/agents/_shared/frameworks.md` criado
2. Cobre 8 frameworks com explicação **≥3 parágrafos cada**: AIDA, PAS, 4 U's (Útil/Urgente/Único/Ultraespecífico), 5 estágios de consciência (Schwartz), 5 níveis de sofisticação de mercado (Schwartz), slippery slide (Sugarman), inherent drama (Burnett), reason-why (Hopkins)
3. Cada framework: definição + quando usar + exemplo concreto + atribuição ao copywriter/livro de origem
4. Total **≥ 2500 palavras** (denso, não stub) — validado por NFR1 (≥600 minimum applied to _shared/) ou critério próprio do _shared/
5. Sumário/index navegável no topo (linkando aos 8 frameworks)
6. Linguagem PT-BR, terminologia técnica em inglês preservada (ex: "headline", "lead")
7. Linkado de `docs/architecture/knowledge-base.md` (criar este arquivo se não existir)
8. **CI passa em verde** após este commit (workflow validate.yml roda sem violations)

- ⚠️ **Gap arquitetural identificado pelo @sm:** `validate-structure.js` (Story 1.3) lista `_shared/` como agent dirs. Mas `_shared/frameworks.md` é knowledge base, não agent. Sem fix, CI vai falhar quando frameworks.md for adicionado (vai tentar validar frontmatter de agent que não existe). **Task 0 corrige isso antes de adicionar o conteúdo.**

### Architectural decision: `_shared/` vs `.claude/agents/`

> **Source:** Architecture §3.1 + §3.5

- `.claude/agents/{nome}.md` — **agent files** com YAML frontmatter de schema agent (name, type, tools, etc.). Validados por `validate-structure.js`.
- `.claude/agents/_shared/{nome}.md` — **knowledge base** compartilhada. NÃO são agents. Validados apenas por `validate-density.js` (NFR1 ≥600 words) e `encoding-check`.

Esta distinção foi implícita em Architecture §3.5 mas não enforced pelo código. Story 1.5 Task 0 enforce.
