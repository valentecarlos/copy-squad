## Story

**As a** copywriter user (Carlos),
**I want** um agente `halbert` invocável via `@halbert` ou pelo Copy Master,
**so that** eu possa pedir copy no estilo Halbert e receber peça **baseada em pesquisa real** (lazy-load de `research/halbert/`), não pastiche generativo.

### Architecture reference — Agent File Format (docs/architecture.md §3.2)

**Schema canônico** (template a seguir):

```markdown
---
name: halbert
description: |
  Use Halbert para copy de direct response cru, emocional, com headlines magnéticas
  e sales letter longa. Mercados: saúde, dinheiro, relacionamentos, info-produto.
tools: [Read, Grep]
type: copywriter
specialty: direct-response
research_path: research/halbert/
language: pt-BR
---

# Halbert (Gary Halbert) — The Prince of Print

## 1. Identidade
[≥80 palavras]

## 2. Voz e Estilo
[≥120 palavras]

## 3. Frameworks que Domina
Referenciar `research/halbert/frameworks.md` e `.claude/agents/_shared/frameworks.md`.
[≥100 palavras]

## 4. Peças de Referência
Referenciar `research/halbert/exemplos.md` para texto integral.
[≥80 palavras]

## 5. Como Você Opera no Squad
[≥100 palavras]

## 6. Regras Duras
- Português brasileiro nativo (NFR9)
- Nunca quebra o personagem
- Cita peça/fonte ao aplicar técnica específica
- Devolve sempre 2 versões + nota da aposta
[≥120 palavras]
```

**Total mínimo:** ≥600 palavras de body.

### Validation scripts já existentes

- `validate-density.js` — checa NFR1 (agent body ≥600 words) + NFR2 (research dirs ≥3500)
- `validate-structure.js` — checa YAML frontmatter (required fields, tools whitelist, name único, research_path FR5, language pt-BR NFR9)
- `validate-research-format.js` — checa DEC-002 sources schema em research/* (não applies a agents)

Esses scripts **já estão configurados em CI**; agent novo será automaticamente validado.

### Mantras canônicos Halbert a incluir

- **"Motion beats meditation"** — agir vence planejar excessivamente
- **"The Boron Letters"** — autoridade book mentioned
- **"Starving Crowd"** — framework central (mercado faminto > produto perfeito)
- **"A-pile vs B-pile"** — separar peças que parecem mass-mail vs. carta pessoal
- **"Prince of Print"** — apelido autodeclarado, parte da brand pessoal
- **"What's the offer?"** — pergunta de filtro que Halbert ensina

### Description distintivo (quando invocar Halbert)

Halbert deve ser invocado quando o brief envolve:
- **Direct response cru, emocional**
- **Headlines magnéticas + sales letter longa**
- **Mercados:** saúde, dinheiro, relacionamentos, info-produto
- **Voz "amigo no bar com edge"** (vs. corporate, vs. literary, vs. formal)
- **Tom de underground marketing** + autenticidade vs. polimento corporativo

Diferenciar de outros DR copywriters (que será documentado em Stories 3.2-3.5).

### Word count target

- **Min:** ≥600 palavras body (NFR1)
- **Recomendado:** 800-1.000 palavras (consistent com Halbert dossier density level)
- **Max:** sem limit explicit — qualidade > quantidade, mas avoid bloat

### Technical Constraints

- **Frontmatter parseable** — YAML válido (sem syntax errors)
- **Tools whitelist** — apenas `Read, Grep` (terminal subagent — sem Agent permission, anti-recursion)
- **Encoding** UTF-8 + LF + sem BOM
- **Sem novas deps** — apenas js-yaml já instalado
- **PT-BR** narrative (NFR9)
